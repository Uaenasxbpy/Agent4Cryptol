参数使用规则：

该项目最近的编译器反馈：
- 生成的 Cryptol 中 `shiftR` 不在作用域内
- 生成的 Cryptol 中 `(&)` 不在作用域内
- 生成的 Cryptol 中 `fromIntegral` 不在作用域内
- `(n : Integer, lgw : Integer) -> Integer` 会导致解析错误
- 把类型参数 `n` 当作运行时值会导致 `Expected a value named 'n' but found a type instead`。修复方式：使用反引号 `` `n `` 将类型降格为值
- `[Integer][8]` 这种序列类型写法是非法的，因为长度必须是数值类型表达式，而不能是值类型 `Integer`
- `[0 .. `n - 1]` 会导致 `The boundaries of .. sequences should be valid numeric types`。修复方式：使用 `take`{n} [0 ...]` 或类型级边界 `[0 .. (n-1)]`（不要用反引号）
- 混用不同位宽的比特向量（例如 `[7]` + `[9]`）会导致 `Type mismatch: Expected type 9, Inferred type 7`。修复方式：使用 `zext` 扩展较窄的值
- `div` 和 `mod`（以及 `/` 和 `%`）是合法的 Cryptol 运算符，应自由用于模运算
